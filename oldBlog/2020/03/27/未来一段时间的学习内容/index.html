未来一段时间的学习内容

发表于 2020-03-27 更新于 2020-12-03 分类于 随笔
本文字数： 1.7k 阅读时长 ≈ 3 分钟

最近课程陆陆续续都开了。编程的作业也渐渐多了起来。

最近买了几本书，准备学一些编程语言之外的东西了。照片如下。

甲.《大话数据结构》

这是一本数据结构的入门书。为了提高程序运行效率，写出好程序，你必须要了解数据结构的一些概念，来优化你的程序逻辑。比如排序、索引，排队这样的问题，当样本大了，用传统的思想去做恐怕运行效率很低，所以有必要去了解一些巧的方法达到事半功倍的目的。

乙.《Effective C++》

这本书可以看作是C++编程的进阶材料和标准材料。这本书的目的是告诉你如何有效运用C++编程。起码，我觉得这是一本很有用的参考书。书中详细介绍了55条C++编程的注意事项（正像题目所说：55个改善程序与设计的55个具体做法）。有些条款看似不解，但是认真看完觉得受益匪浅。如第二条“尽量以const,enum,inline替换#define”，#define我们平时很常用，但是细细看完你就不想再用#define了。
就像书中的各位读者的评语说的那样：“这本书萃取了原本必须历经艰辛才能学到的编程经验。”我想，这本书一定能让你重新认识C++，如果你励志要学好C++的话。

丙.《编码》

买这本书来源于老刘的一篇微信推文假如我是计算机系老师。

假如我是计算机系老师， 我会先让我的学生先去读一下《编码：隐匿在计算机软硬件背后的语言》这本书， 搞明白一个个简单的逻辑门电路到底是怎么组成计算机的， 弄清楚计算机软件语言和硬件之间到底是怎么关联起来的，然后再去上计算机组成原理，模拟电路，数字电路等专业课， 你会发现境界不同了，你会开启“上帝视角”进行俯视， 只需要通过这些专业课程补全知识的细节就可以了。

正如上述，这一本讲述计算机原理的书，但是并不枯燥，内容十分科普十分幽默。从莫斯电码讲到二进制，从电报机讲到逻辑电路，再到微处理器、操作系统，看得我都想亲手打造一个二进制加法器🤪。

这确实对了解计算机原理有很好的代入作用。你会知道int float 的8位、32位是什么意思；CPU的32位和64位是怎么来的，CPU是怎么做加法的等等等等。

丁.《编程珠玑》

这本书也是讲数据结构算法的，比上边的《大话数据结构》要更具体。

在《大话数据结构》没看完之前，这本书现放一放。

放一段吴师兄的介绍语。全文见初学算法该直接上《算法导论》 还是先看完 《算法 第四版》？

《编程珠玑》这本书的豆瓣评分非常高，有 9 分。

这本书最大的特色就是讲了很多针对海量数据的处理技巧。这个可能是其他算法书籍很少涉及的。面试的时候，海量数据处理的问题也是经常会问的，特别是校招面试。不管是开拓眼界，还是应付面试，这本书都很值得一看。

言而总之，我要进阶了。希望以后的文章更加有水平。

更少错别字🤪

题外话

最近我身边的一些朋友都在我的“蛊惑”之下开始学Python了，也是我建议他们学的。编程语言孰优孰劣暂且不说（说也说说不清楚），给他们推荐Python，重要的原因就是我不想让他们在想学编程的时候彻底丧失对编程的兴趣。

Python 我也接触过，写过一些爬虫的项目。不过我却没有想法去更深入的学习Python，尤其是写过几个Qt的GUI程序后，被C++深深的吸引了。虽然Python也有类，处处是对象，但是知道Python的解释器也是用C写的（当然也有Java写的），Python的很多库也是用C++写的，就对其失去兴趣了。说白了Python就是一个套壳（这么说有失偏颇）。。不过不可否认其使用方便。

顺便一提，为什么说Python在人工智能、机器学习方面有优势，成为了“宠儿” ？它真的是有魔力吗？实际上，就是因为其接口方便，外部库多（前人给你铺过路了），我们知道人工智能需要大量的数学知识、概率知识，显然用C/C++来做是非常考验研究者的，不仅要有深厚的数学功底还要熟悉掌握C/C++的语法，尤其是处理矩阵、概率的方法，显然这不是大多数研究者乐意的。所以一门简单强大的语言受宠也就很合理了。

但这并不是说C/C++就淘汰了、过时了。只能说各有千秋。

换一个角度开看，与其纠结在各种语言语法之间徘徊，不如学一些数据结构与算法的思想。语言归根到底是一种工具。
