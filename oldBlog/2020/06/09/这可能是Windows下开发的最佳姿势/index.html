这可能是Windows下开发的最佳姿势

发表于 2020-06-09 更新于 2020-12-08 分类于 随笔
本文字数： 905 阅读时长 ≈ 2 分钟

平时我是用Linux做主力（开发+日常），偶尔用到一些Windows平台独占应用（如OneNote、ArcGIS等）都是用虚拟机解决的。

不过我最近又装上了Windows 10.

原因如下：

a. 随着Windows 10 2004 的推出，带来了更好的WSL2（Windows Subsystem for Linux 2 适用于windows的Linux子系统），一定程度上解决了我对Linux桌面版的依赖，(我使用Linux主要是为了开发和非常高效的bash命令)

b. Windows 10 LSTC在笔记本上运行非常稳定，风扇几乎不转，但是没有应用商店，没有Linux子系统。就是开发配置没有Linux快，这方面Linux处于不可替代的高度

c. 舍不得Onedrive、OneNote、Ofiice。虽然Linux也由替代品，如坚果云、有道云笔记、WPS等，但是你要最好的体验都得付费，也不是一家公司，要分别付费

我认为的最佳姿势

我用过的几种编译姿势.

  • Windows
    • VS201x+cl
    • CLion+mingw64-gcc
    • CLion+cl
    • CLion+WSL x

我现在的配置

  • 编辑器：VSCode
  • 编译器：VS Build Tool（cl）
  • 库管理器：vcpkg
  • 项目配置：CMake
  • 项目管理：git

展望——Window和Linux

随着国家信息产业的不断发展，Linux肯定是未来，现在的一些所谓的国产操作系统（Deepin、中标麒麟、统信OS等），无一例外都是基于Linux内核制作的，加入Windows真的不给中国市场使用了（我觉得可能性很小），Linux将在国内市场普及。即便不会出现Windows禁用的事件，发展自己的操作系统也是必要的。我可以不用，但不能没有。

现在Linux软件生态还不是很好，国内的常用很多应用也没有Linux版本。常用的应用越少，普通老百姓用户越少、公司越不想花精力开发Linux版，这样导致一个恶行循环。

不过随着国家的复兴，信息核心产业（操作系统、芯片等）都会成为国家鼓励支持的对象，尤其是美国的封锁“助攻”，更是让大家知道了自主制造的重要性。所以我觉得未来Linux即便在中国不会完全取代Windows，但它也占有很大的市场。
